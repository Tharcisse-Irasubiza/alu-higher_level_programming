# python-test_driven_development

This project covers Test-Driven Development in Python: writing functions
with strict input validation, documenting them with docstrings, and
verifying behavior with both `doctest` (interactive-style tests in `.txt`
files) and `unittest` (structured test classes).

## Files

- `0-add_integer.py` — adds two integers/floats, with type validation
- `2-matrix_divided.py` — divides all elements of a matrix by a number
- `3-say_my_name.py` — prints a formatted full name
- `4-print_square.py` — prints a square of `#` characters
- `5-text_indentation.py` — prints text with extra newlines after `.?:`
- `6-max_integer.py` — finds the max integer in a list
- `tests/0-add_integer.txt` — doctest for add_integer
- `tests/2-matrix_divided.txt` — doctest for matrix_divided
- `tests/3-say_my_name.txt` — doctest for say_my_name
- `tests/4-print_square.txt` — doctest for print_square
- `tests/5-text_indentation.txt` — doctest for text_indentation
- `tests/6-max_integer_test.py` — unittest for max_integer
